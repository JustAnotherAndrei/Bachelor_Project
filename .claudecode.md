# Project Context: Q-Shield (QKD Platform)

## 1. Project Identity & Vision
- **Title:** Q-Shield: O Platformă Distribuită pentru Generarea Cheilor Cuantice (BB84).
- **Core Goal:** O aplicație Full-Stack care permite utilizatorilor să simuleze sau să execute pe hardware real protocolul BB84, analizând impactul zgomotului și al interceptărilor (Eve).
- **Developer:** Andrei (Bachelor's Thesis Project)

## 2. Technical Stack (The "Pro" Stack)
- **Quantum:** Python 3.11+, Qiskit, Qiskit-Aer.
- **Hardware Integration:** IBM Quantum Runtime API (pentru execuție pe calculatoare cuantice reale).
- **Backend:** FastAPI (Python) cu WebSockets pentru comunicare asincronă în timp real.
- **Frontend:** React 18+ (Vite), Tailwind CSS (UI modern), Lucide Icons, Recharts (grafice QBER).
- **Infrastructure:** Docker & Docker Compose pentru containerizarea completă (Backend, Frontend, Nginx).
- **Documentation:** LaTeX (Overleaf) - stil academic formal.

## 3. Project Architecture
- `/backend`:
    - `quantum_logic/`: Generare circuite, NoiseModel configurabil, integrare IBM Quantum.
    - `classical_processing/`: Sifting, calcul QBER, Error Correction (Parity blocks), Privacy Amplification (SHA-256).
    - `api/`: Endpoint-uri REST și manager de WebSockets pentru sesiuni Alice-Bob.
- `/frontend`:
    - Componente React pentru Dashboard, vizualizarea "fotonilor", tabele de baze și grafice de eroare.

## 4. Quantum Constraints & Rules
- **Noise Model:** Trebuie să includă `depolarizing_error` și `pauli_error` (measurement error).
- **Security Threshold:** Dacă QBER >= 11%, protocolul trebuie marcat ca "Compromis/Insecure".
- **Hybrid Mode:** Utilizatorul trebuie să poată alege între "Simulator" și "Real IBM Hardware".
